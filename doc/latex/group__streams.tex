\doxysubsection{Detailed Description}
Support for I/O streams. 

The stream model of tinyos3 is similar to the Unix model. Streams are objects that are shared between processes. Streams are accessed by file IDs (similar to file descriptors in Unix).

The streams of each process are held in the file table of the PCB of the process. The system calls generally use the API of this file to access FCBs\+: \mbox{\hyperlink{group__streams_gab653c475f793598f10c15d4e2dd900da}{get\+\_\+fcb}}, \mbox{\hyperlink{group__streams_ga462269376de145171b87b7bc3036e4f8}{FCB\+\_\+reserve}} and \mbox{\hyperlink{group__streams_gac44c094845a8d4e2e13f9df5b17274df}{FCB\+\_\+unreserve}}.

Streams are connected to devices by virtue of a {\ttfamily \mbox{\hyperlink{structfile__operations}{file\+\_\+operations}}} object, which provides pointers to device-\/specific implementations for read, write and close.

Acquire a number of FCBs and corresponding fids. 

Given an array of fids and an array of pointers to FCBs of size @ num, this function will check is available resources in the current process PCB and FCB are available, and if so it will fill the two arrays with the appropriate values. If not, the state is unchanged (but the array contents may have been overwritten).

If these resources are not needed, the operation can be reversed by calling \mbox{\hyperlink{group__streams_gac44c094845a8d4e2e13f9df5b17274df}{FCB\+\_\+unreserve}}.

Release a number of FCBs and corresponding fids. 

Given an array of fids of size @ num, this function will return the fids to the free pool of the current process and release the corresponding FCBs.

This is the opposite of operation \mbox{\hyperlink{group__streams_ga462269376de145171b87b7bc3036e4f8}{FCB\+\_\+reserve}}. Note that this is very different from closing open fids. No I/O operation is performed by this function.

This function does not check its arguments for correctness. Use only with arrays filled by a call to \mbox{\hyperlink{group__streams_ga462269376de145171b87b7bc3036e4f8}{FCB\+\_\+reserve}}.
